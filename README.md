# 🥛 Milk Diary

A warm, intuitive, and simple progressive web application built with love for my mom to easily track our family's daily milk consumption.

## 💝 Why This Was Built

Our family gets milk delivered daily by a vendor. Usually, it's 1 liter a day, but this varies—sometimes we take extra for guests, and sometimes we skip days when we're away. 

For years, my mom managed this by marking a physical calendar on the wall with the daily quantities, and at the end of every month, she'd manually calculate the total liters and multiply it by the price per liter to pay the vendor. 

I built **Milk Diary** to completely automate this process for her. It replaces the physical calendar with a beautiful, iPad-optimized digital calendar that does the math automatically.

## ✨ Features

- **📅 Interactive Calendar View:** Tap any date to quickly log milk quantities or mark a day as "Skipped".
- **💰 Automatic Math:** The app automatically calculates the total liters consumed and the grand total due for the month based on a customizable price-per-liter setting.
- **⚡️ One-Tap Logging:** Quick presets (250ml, 500ml, 1L, etc.) plus a custom amount input makes logging faster than writing it down.
- **💾 Auto-Saves Locally:** No accounts, no logins, no cloud syncing needed. Everything is securely saved directly on the device using browser `localStorage`.
- **📱 iPad Optimized & Responsive:** Designed with large, friendly touch targets, warm cozy colors, and a full-screen layout that responds beautifully to any device orientation.

## 🛠️ Technology Stack

- **Vanilla HTML5**
- **Vanilla CSS3** (Flexbox/Grid, Custom CSS Variables)
- **Vanilla JavaScript** (ES6+)
- **Zero Dependencies:** No frameworks or external libraries to ensure it loads instantly, works offline, and requires zero long-term maintenance.

## 🚀 How to Use

Simply open `index.html` in any web browser! 

For the best experience on mobile or tablet (like an iPad), open the file in Safari and select **"Add to Home Screen"**. It will behave exactly like a native app.
